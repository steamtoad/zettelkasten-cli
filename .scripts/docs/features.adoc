== Назначение

Документ описывает подтверждённую текущую функциональность `zettelkasten-cli`.
В список не включаются планы и предполагаемое поведение, которое не подтверждено реализацией или проверкой.

== Создание заметок

* `CREATE-001` — создание обычной Note через `zt-note.zsh`.
* `CREATE-002` — создание датированного Memo через `zt-memo.zsh`.
* `CREATE-003` — создание датированного Todo с первой пустой задачей через `zt-todo.zsh`.
* `CREATE-004` — создание Topic через `zt-keytopic.zsh`.
* `CREATE-005` — создание дневниковой записи через `zt-diary.zsh`.
* `CREATE-006` — генерация UUID v1 / time-based UUID для имени нового файла.
* `CREATE-007` — генерация UUID на Linux через `uuidgen -t` и на macOS через `uuid`.
* `CREATE-008` — интерактивный запрос названия для Note, Memo, Todo, Topic и новой Topic в режиме Clean Topic.
* `CREATE-009` — отмена создания при пустом обязательном названии.
* `CREATE-010` — автоматическое открытие созданного постоянного документа в Vim.
* `CREATE-011` — вывод готовой AsciiDoc-ссылки после создания постоянного документа.
* `CREATE-012` — автоматическая регистрация созданного постоянного документа в `all-todays`.
* `CREATE-013` — автоматическое создание каталога `all-todays` при необходимости.
* `CREATE-014` — создание Memo с опциональной привязкой к активной Topic.
* `CREATE-015` — создание Note с опциональной привязкой к активному Memo.

== Типы документов

* `TYPE-001` — обычная заметка: `:type: note`.
* `TYPE-002` — Memo: `:type: memo`.
* `TYPE-003` — список задач: `:type: todo`.
* `TYPE-004` — дневниковая запись: `:type: diary`.
* `TYPE-005` — ключевая тема: `:type: topic`.
* `TYPE-006` — генерируемое поисковое представление в stdout: `:type: index`.
* `TYPE-007` — генерируемый листинг содержимого в stdout: `:type: list`.
* `TYPE-008` — использование `:key-topic:` как ключа тематической группировки и связи документов.

== Метаданные

* `META-001` — единая генерация заголовка и основных атрибутов постоянных документов через общую библиотеку.
* `META-002` — автоматическая дата создания постоянного документа.
* `META-003` — автоматический автор из `whoami` для новых документов; Full Copy сохраняет автора исходной Topic.
* `META-004` — автоматические `:description:`, `:doclink:` и `:docfilename:` для новых постоянных документов.
* `META-005` — автоматическое назначение `:keywords:` и `:type:`.
* `META-006` — чтение атрибутов AsciiDoc через общую библиотеку.
* `META-007` — игнорирование атрибутоподобных строк внутри поддерживаемых библиотекой блоков кода.
* `META-008` — определение deprecated-документа по наличию атрибута `:deprecated:`.
* `META-009` — наследование `:key-topic:` при привязке, если атрибут присутствует у исходного документа.
* `META-010` — наследование ключевых слов Topic при создании связанного Memo.
* `META-011` — наследование ключевых слов Memo при создании связанной Note.
* `META-012` — исключение тега непосредственного исходного типа: `topic` при Topic→Memo и `memo` при Memo→Note.
* `META-013` — получение описания ссылки через последовательный fallback: `:description:`, заголовок документа, имя файла.

== Дневник

* `DIARY-001` — хранение имени последней дневниковой записи в `.last-diary`.
* `DIARY-002` — ссылка новой дневниковой записи на предыдущую.
* `DIARY-003` — добавление ссылки на следующую запись в предыдущий дневник.
* `DIARY-004` — формирование двунаправленной временной цепочки дневников.
* `DIARY-005` — регистрация новой дневниковой записи в `all-todays`.

== Поиск и навигация

* `SEARCH-001` — регистронезависимый полнотекстовый поиск по регулярному выражению через `zt-find.zsh`.
* `SEARCH-002` — вывод результатов поиска как AsciiDoc-представления со ссылками.
* `SEARCH-003` — объединение полного содержимого найденных документов через `zt-read.zsh`.
* `SEARCH-004` — возможность перенаправить stdout `zt-find.zsh` и `zt-read.zsh` в файл.
* `SEARCH-005` — интерактивный выбор документов через `fzf` в навигации и специализированных селекторах.
* `SEARCH-006` — отображение документов в интерактивном выборе по `:description:`.
* `SEARCH-007` — открытие выбранного документа в Vim через `zt-edit.zsh`.
* `SEARCH-008` — генерация ссылки на выбранный документ через `zt-getlink.zsh`.
* `SEARCH-009` — исключение deprecated-документов из поиска, навигации и специализированных селекторов.
* `SEARCH-010` — фильтрация специализированных селекторов по требуемому `:type:`.

== Связи между документами

* `LINK-001` — опциональная автопривязка Memo к Topic.
* `LINK-002` — наследование `:key-topic:` при создании Memo, если атрибут присутствует у Topic.
* `LINK-003` — наследование ключевых слов Topic при создании Memo с исключением тега `topic`.
* `LINK-004` — автоматическое создание двусторонних ссылок Topic↔Memo.
* `LINK-005` — поддержка нескольких Memo для одной Topic.
* `LINK-006` — опциональная автопривязка Note к Memo.
* `LINK-007` — наследование `:key-topic:` при создании Note, если атрибут присутствует у Memo.
* `LINK-008` — наследование ключевых слов Memo при создании Note с исключением тега `memo`.
* `LINK-009` — автоматическое создание двусторонних ссылок Memo↔Note.
* `LINK-010` — защита от дублирования ссылки по UUID целевого документа во всём исходном документе.
* `LINK-011` — автоматическое создание раздела связей, если раздел с точным требуемым заголовком отсутствует.
* `LINK-012` — поддержка построения опциональной цепочки Topic→Memo→Note.
* `LINK-013` — автоматическое создание двусторонних ссылок между старой и новой Topic при Reduce.
* `LINK-014` — использование меток `Развитие:` и `Основано на:` для связи поколений Topic.
* `LINK-015` — поддержка цепочки нескольких поколений Topic.
* `LINK-016` — идемпотентное добавление связей по UUID целевого документа.

== Reduce

* `REDUCE-001` — выполнение Reduce для выбранной активной Topic.
* `REDUCE-002` — выбор активной Topic через `fzf`.
* `REDUCE-003` — выбор режима создания новой Topic после выбора исходной Topic.
* `REDUCE-004` — Full Copy создаёт новую Topic на основе старой, обновляет `:doclink:` и `:docfilename:` и удаляет у копии `:deprecated:`.
* `REDUCE-005` — Clean Topic создаёт новую Topic с введённым пользователем названием, базовыми метаданными и унаследованным `:key-topic:`.
* `REDUCE-006` — сохранение точного значения `:key-topic:` независимо от режима.
* `REDUCE-007` — выбранная старая Topic помечается как deprecated.
* `REDUCE-008` — активные Memo с точным совпадением `:key-topic:` помечаются как deprecated.
* `REDUCE-009` — Note никогда автоматически не помечаются как deprecated.
* `REDUCE-010` — активные Note с точным совпадением `:key-topic:` связываются с новой Topic.
* `REDUCE-011` — при повторном Reduce активные Note связываются с очередным поколением Topic.
* `REDUCE-012` — повторный Reduce архивирует только выбранную активную Topic и создаёт её преемника; глобальная уникальность активной Topic не обеспечивается.
* `REDUCE-013` — старые Topic сохраняются как архивные документы.
* `REDUCE-014` — обработанные Memo сохраняются как архивные документы.
* `REDUCE-015` — существующие UUID, имена файлов и ссылки не переименовываются.
* `REDUCE-016` — deprecated Memo и Note исключаются из обработки.
* `REDUCE-017` — Memo и Note с другим `:key-topic:` не изменяются.
* `REDUCE-018` — Full Copy изменяет метаданные только в заголовке и буквально сохраняет тело документа и блоки исходного кода.
* `REDUCE-019` — новая Topic регистрируется в `all-todays`.
* `REDUCE-020` — после выполнения выводятся старая и новая Topic, количество архивированных Memo и связанных Note.

== Проверка целостности

* `CHECK-001` — проверка существования основных элементов структуры репозитория.
* `CHECK-002` — проверка обязательных метаданных постоянных документов в корне Zettelkasten.
* `CHECK-003` — проверка наличия заголовков.
* `CHECK-004` — проверка известных значений `:type:`.
* `CHECK-005` — рекомендация добавить значение `:type:` в `:keywords:`.
* `CHECK-006` — проверка соответствия `:docfilename:` имени файла.
* `CHECK-007` — проверка корректности `:doclink:`.
* `CHECK-008` — поиск битых ссылок между документами.
* `CHECK-009` — проверка ссылок из `all-todays`.
* `CHECK-010` — проверка `.last-diary`.
* `CHECK-011` — проверка существования документов, указанных в diary-цепочке.
* `CHECK-012` — игнорирование ссылок внутри блоков `----` и Markdown-блоков с тройными обратными кавычками.
* `CHECK-013` — раздельный вывод ошибок и рекомендаций.
* `CHECK-014` — ненулевой код завершения при наличии ошибок.
* `CHECK-015` — проверка без автоматического изменения файлов.
* `CHECK-016` — исключение placeholder-ссылок `UUID.adoc`, `previous.adoc` и `next.adoc` из проверки битых ссылок.

== Обслуживание и инфраструктура

* `MAINT-001` — исправление отсутствующего завершающего LF в CLI-скриптах через `zt-scripts-patch.zsh`.
* `MAINT-002` — вывод всех CLI-скриптов и библиотек для ревью через `zt-scripts-review.zsh`.
* `MAINT-003` — вывод всех репозиторных навыков `zt-*.adoc` для ревью через `zt-skills-review.zsh`; индекс `README.adoc` не включается.
* `MAINT-004` — `zt-skills-review.zsh` использует `ZK_HOME`, выводит read-only секции с разделителями и именами файлов и завершается с ошибкой, если `.agent-skills` отсутствует.
* `CORE-001` — общие библиотеки для путей, UUID и AsciiDoc.
* `CORE-002` — поддержка `ZK_HOME` большинством скриптов.
* `CORE-003` — локализованные русские приглашения, сообщения и заголовки.
* `CORE-004` — использование zsh как языка CLI-скриптов.

== Известные ограничения

* `zt-scripts-patch.zsh` использует жёсткий путь `$HOME/zettelkasten/.scripts` и не поддерживает `ZK_HOME`.
* Исключение служебных тегов при наследовании ограничено тегом непосредственного исходного типа.
* Проверка ссылок внутри блоков кода в `zt-check.zsh` распознаёт только блоки `----` и Markdown-блоки с тройными обратными кавычками.
* Reduce не обеспечивает глобальную уникальность активной Topic по `:key-topic:`.
* Скрипты доступны как файлы `.scripts/zt-*.zsh`; команды без расширения не устанавливаются в `PATH`.
